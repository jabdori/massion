# Phase 25 — 모델 평가실과 역할별 자동 배치 설계

> **상태**: approved
> **승인일**: 2026-07-12
> **상세 설계**: 새 Core 저장소 전환 뒤 Phase 25 시작 commit에서 작성
> **선행 단계**: Phase 24 구현·로컬 검증 완료 및 Phase 27의 새 Massion Core root commit 생성. Phase 24 외부 계정 UAT는 이 Phase와 함께 종료하는 별도 조건입니다.

## 1. 목적

사용자가 연결한 구독·API·로컬 모델을 역할별로 평가하고, 품질·비용·속도·개인정보 정책에 맞는 주 모델과 순서가 있는 대체 모델(fallback model)을 추천합니다. 최초 추천은 사용자가 승인해야 하며, 자동 최적화는 조직별 명시적 동의가 있을 때만 제한된 승격과 자동 복구를 수행합니다.

## 2. 제품 경계

- 사용자가 연결하지 않은 무료·공개 모델은 내장하거나 자동 호출하지 않습니다.
- 모델 평가실(Model Evaluation Lab)은 역할별 장기 적합성을 판단하고, 기존 모델 라우터(Model Router)는 요청 시점의 계정 상태·할당량·회로 차단·fallback을 판단합니다.
- 평가 점수는 전역 단일 수치가 아니라 역할·평가 묶음·프롬프트·도구·런타임 버전을 포함한 불변 영수증(receipt)으로 보존합니다.
- 실제 작업 관찰(Production Learning), 그림자 실행(shadow run), 자동 승격은 각각 기본 비활성화하며 조직 정책의 명시적 동의를 요구합니다.
- 그림자 실행은 Work, 파일, 메시지, 배포, 승인 소비와 조직 정본을 변경할 수 없습니다.

## 3. 요구사항

- `REQ-OPTIMIZATION-001`: 코어 역할 8개와 소프트웨어 개발 실행 역할 8개에 버전이 있는 평가 묶음과 자격 기준을 제공합니다.
- `REQ-OPTIMIZATION-002`: 하드 게이트 뒤에 최고 품질·가성비·속도·개인정보 우선·수동 고정 정책을 결정론적으로 적용하고 주 모델과 fallback을 추천합니다.
- `REQ-OPTIMIZATION-003`: 최초 추천 승인, 선택적 자동 최적화, 최소 표본·개선 폭·그림자 실행·제한 배치·원자적 승격·자동 복구를 계보와 함께 제공합니다.
- `REQ-OPTIMIZATION-004`: CLI·TUI·Web과 설치 release에서 평가·추천·적용·복구를 제공하고 Claude·Codex·GLM 실제 연결 시나리오를 `tmux`로 검증합니다.

## 4. 완료 조건

- 상세 설계 16절의 완료 조건을 모두 충족합니다.
- Phase 24의 계정 회전·할당량·fallback과 기존 Core Office·Software Engineering 경로에 회귀가 없습니다.
- 자동 최적화 비활성 조직에는 실사용 관찰과 그림자 실행이 생성되지 않습니다.
- 비밀·개인정보 노출과 미해결 CRITICAL·MAJOR finding이 0이며 실행하지 못한 외부 계정은 명시합니다.
- 전체 품질·보안·복구·설치 릴리스 검증을 다시 통과합니다.
